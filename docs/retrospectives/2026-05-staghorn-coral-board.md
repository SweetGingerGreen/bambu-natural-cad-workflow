# 2026-05 鹿角蕨珊瑚板建模复盘

## 背景

用户想做鹿角蕨上板，参考了 Bambu Studio 中一个珊瑚状圆盘模型，并通过 GPT Image 生成多组方案。用户最终明确选择 **E2 方案**：外部为珊瑚盾牌/海扇状镂空，中心为圆盘状水苔固定区。

本次迭代最后有效文件：

- `output/stl/staghorn_e2_clean_v3.stl`
- `output/preview/staghorn_e2_clean_v3_mask.png`
- `scripts/trace_e2_staghorn_clean.py`

## 主要问题

### 1. 没有严格遵循用户选定的设计图

用户说 “E2 好看，生成打印文件” 后，第一次实现是手写参数化近似版，只借用了 “珊瑚 + 圆盘” 的概念，并没有严格复刻 E2 的外轮廓、孔洞节奏和整体比例。

结果：

- 外轮廓和 E2 明显不一样；
- 孔洞分布更像普通放射网格；
- 用户需要再次指出 “严格按照 E2 图中的造型来建模”。

后续规则：

- 当用户明确选中某个图例编号（如 A/C/E2）时，默认目标是 **按图复刻**，不是自由发挥。
- 在开始建模前先声明建模方法：
  - 严格复刻：图像追踪 / 手工矢量描摹 / 按图参数化；
  - 风格借鉴：只保留视觉语言，允许重设计。
- 如果做不到严格复刻，必须先说明限制，不要把 “受启发版本” 当成交付件。

### 2. 早期参数化版本偏离审美目标

第一版圆盘和后续叶形版都试图用程序规则生成孔洞，但视觉结果偏机械：

- 圆盘版像普通海绵垫，不像鹿角蕨板；
- 叶形版孔洞过于横向，用户评价像 “大蟑螂”；
- 强度有所提升，但审美方向错误。

原因：

- 只从结构可打印性出发，没有先锁定用户认可的视觉轮廓。
- 把 “叶形” 理解成普通叶片，而不是用户后来选中的 E2 珊瑚盾牌形。

后续规则：

- 装饰件先解决视觉方向，再解决工程细节。
- 用户强烈否定造型时，不要继续局部修补；应回到图例确认或重新生成方案。
- 对这类板状装饰件，先让用户确认 2D 轮廓和孔洞节奏，再挤出厚度。

### 3. 抠图追踪没有先做质量关卡

第一次追踪 E2 时直接从渲染图抠图，虽然比手写版更接近 E2，但把图片中的阴影、高光和边缘抗锯齿也带进了几何。

结果：

- 外轮廓出现锯齿和局部缺口；
- 孔洞边缘不够干净；
- 生成物更像 “从图片硬抠出来的网格”，不够像干净 CAD。

后续规则：

- 图像追踪后必须先输出并查看 `mask.png`，确认：
  - 外轮廓是否干净；
  - 大孔洞是否完整；
  - 阴影/文字/背景是否被误识别；
  - 中心功能区是否清楚。
- `mask.png` 没通过，不允许直接导 STL 打开给用户看。
- 对 GPT Image 渲染图，不能直接信任像素边缘；需要做几何平滑或手工重建关键区域。

### 4. 中心圆盘被叠加了两次

第一次 “干净版” 仍然有中心问题：原图追踪里保留了一个旧圆盘边界，又额外 `union` 了一个新的圆盘。两个圆心略有偏差，Bambu 中看起来像两个圆盘交叠。

原因：

- 只检查了 STL 是否单一组件，没有检查中心区域的视觉语义。
- 用 “加一个新圆盘” 修正中心，而不是先删除旧中心再替换。

最终修正：

- 先从追踪多边形中挖掉中心清理区；
- 再放回唯一一个圆盘；
- 绑线孔从这个唯一圆盘中贯穿减掉；
- 用 `staghorn_e2_clean_v3_mask.png` 确认中心只剩一个圆盘。

后续规则：

- 功能区必须做 “替换”，不要做 “叠加修补”。
- 中心圆盘、挂孔、接口区这类关键结构要用干净几何重建，不能完全依赖图片抠图。
- 网格检查不能只看 `components=1`，还要看视觉上是否有重叠边界。

### 5. 没有把 “参考图复刻” 和 “可打印结构” 分成两层

E2 是渲染概念图，本身不是 CAD 文件。直接照图生成时，需要区分两层：

- 视觉层：外轮廓、孔洞节奏、中心圆盘比例；
- 工程层：厚度、最小筋宽、挂孔尺寸、绑线孔直径、平背打印。

本次前几版把两层混在一起，导致要么视觉偏离，要么抠图太脏。

后续规则：

- 先从图里提取视觉层，得到干净 2D mask。
- 再用工程参数重建关键结构：
  - 板厚建议 8-12mm；
  - 最小筋宽至少 5-6mm；
  - 绑线孔建议 3.5-4.5mm；
  - 顶部挂孔必须保留足够肉厚；
  - 背面保持平面，优先无支撑打印。

### 6. Bambu Studio 预览仍需严格隔离版本

本次多次打开参考 3MF、手写版、追踪版、干净版。过程中 Bambu Studio 多次弹出 restore 提示，存在恢复旧项目或误叠旧稿的风险。

后续规则：

- 打开新版前先退出 Bambu Studio；
- 出现 restore 弹窗时，默认点 Cancel，不恢复旧项目；
- 当前要给用户看的 STL 必须在回复里说清楚文件名；
- 被用户否定的草稿 STL 应及时从 `output/stl` 删除，避免误开。

## 以后做 “按图生成打印件” 的检查清单

1. 明确用户要的是：
   - 严格按图；
   - 参考风格；
   - 重新设计。
2. 如果是严格按图，先生成或提取 2D 轮廓预览，不直接做 STL。
3. 对图像追踪任务，必须保存并查看 `output/preview/*mask.png`。
4. 中心圆盘、挂孔、螺孔、接口区必须用干净几何重建。
5. 修正局部结构时优先 “删除旧结构再替换”，不要叠加。
6. STL 检查至少包括：
   - 单一组件；
   - 尺寸符合 Bambu 板面；
   - z 最低点为 0；
   - 视觉上没有重叠边界。
7. 打开 Bambu 前清理旧草稿，并关闭/新建项目。

## 本次有效结论

对鹿角蕨板这类装饰功能件，最稳的工作流不是直接从 AI 图硬抠 STL，也不是完全手写近似，而是：

1. 用户确认图例；
2. 提取或描摹外轮廓和孔洞节奏；
3. 用代码重建中心圆盘、挂孔、绑线孔等功能几何；
4. 导出前先看 mask；
5. 再导 STL 和 Bambu 预览。
